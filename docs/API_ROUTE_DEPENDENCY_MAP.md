# API wired route dependency map

This maps currently wired route dependencies as:

`endpoint -> type -> hook -> UI usage`

| Endpoint                          | Type(s) (`src/lib/api/domains/types.ts`)    | Hook (`src/lib/query/hooks.ts`)   | UI usage (`app/(app)/**`)                                                                                                                                                                     |
| --------------------------------- | ------------------------------------------- | --------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `POST /search`                    | `SearchRequest`, `SearchResponse`           | `useSearchMutation`               | `app/(app)/search/page.tsx` runs search submit flow.                                                                                                                                          |
| `POST /search/save-alert`         | `SaveSearchAlertRequest`, `WatchRule`       | `useSaveSearchAlertMutation`      | `app/(app)/search/page.tsx` saves alert from latest query.                                                                                                                                    |
| `GET /watch-rules`                | `WatchRulesListParams`, `WatchRule[]`       | `useWatchRulesQuery`              | `app/(app)/alerts/page.tsx` renders active alerts list.                                                                                                                                       |
| `GET /watch-rules/:id`            | `WatchRule`                                 | `useWatchRuleDetailQuery`         | `app/(app)/alerts/[id]/AlertDetailClient.tsx` loads a specific alert.                                                                                                                         |
| `POST /watch-rules`               | `WatchRuleCreate`, `WatchRule`              | `useCreateWatchRuleMutation`      | `app/(app)/alerts/new/page.tsx` creates a new alert.                                                                                                                                          |
| `PATCH /watch-rules/:id`          | `WatchRuleUpdate`, `WatchRule`              | `useUpdateWatchRuleMutation`      | `app/(app)/alerts/[id]/AlertDetailClient.tsx` updates alert state/details.                                                                                                                    |
| `DELETE /watch-rules/:id`         | none (`void`)                               | `useDeleteWatchRuleMutation`      | `app/(app)/alerts/[id]/AlertDetailClient.tsx` deletes the alert.                                                                                                                              |
| `GET /watch-releases`             | `WatchReleasesListParams`, `WatchRelease[]` | `useWatchReleasesQuery`           | `app/(app)/alerts/page.tsx` and `app/(app)/watchlist/page.tsx` render matched releases.                                                                                                       |
| `GET /notifications`              | `NotificationsListParams`, `Notification[]` | `useNotificationsQuery`           | `app/(app)/notifications/page.tsx` shows notification feed.                                                                                                                                   |
| `GET /notifications/unread-count` | `NotificationUnreadCount`                   | `useUnreadNotificationCountQuery` | `app/(app)/notifications/page.tsx` displays unread badge/count text.                                                                                                                          |
| `POST /notifications/:id/read`    | none (`void`)                               | `useMarkNotificationReadMutation` | `app/(app)/notifications/page.tsx` marks first unread item read.                                                                                                                              |
| `GET /me`                         | `MeProfile`                                 | `useMeQuery`                      | `app/(app)/settings/profile/page.tsx`, `app/(app)/settings/alerts/page.tsx`, `app/(app)/settings/danger/page.tsx`, and `app/(app)/alerts/new/page.tsx` hydrate user settings/profile context. |
| `PATCH /me`                       | `MeProfileUpdate`, `MeProfile`              | `useUpdateProfileMutation`        | `app/(app)/settings/alerts/page.tsx` persists alert-delivery preferences.                                                                                                                     |
| `DELETE /me`                      | none (`void`)                               | `useDeactivateAccountMutation`    | `app/(app)/settings/danger/page.tsx` deactivates account.                                                                                                                                     |
| `DELETE /me/hard-delete`          | none (`void`)                               | `useHardDeleteAccountMutation`    | `app/(app)/settings/danger/page.tsx` permanently deletes account.                                                                                                                             |
