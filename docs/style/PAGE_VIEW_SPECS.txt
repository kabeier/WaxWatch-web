WAXWATCH PAGEVIEW SPECIFICATIONS (v1)
====================================

Purpose
-------
Defines structure, layout, components, and behavior for each page.
Agents must follow these specs instead of inventing layouts.

------------------------------------------------------------
GLOBAL RULES
------------------------------------------------------------

- All pages live inside AppShell (sidebar + top bar)
- Waveform usage must follow strict rules:
  - Header: calm
  - Dividers: active
  - Special moments: spike (rare)
- Never place waveform behind dense data
- Use cards as primary grouping unit

------------------------------------------------------------
1) DASHBOARD
------------------------------------------------------------

Route:
/
(or /dashboard)

Purpose:
Quick overview of user activity and system state.

Layout:
- Top: optional hero band (subdued WaveTrace active)
- Sections stacked vertically with spacing

Sections:

1. Summary Cards Row
   - Active Alerts count
   - Watchlist count
   - New Matches (last 24h)
   - Unread Notifications
   - Each = card + subtle glow

2. Recent Matches (primary content)
   - Card list
   - Each card:
     - Release name
     - Price
     - Source (eBay/Discogs)
     - Time found
     - CTA: View / Buy
   - Sorted by newest
   - Empty state uses WaveTrace small

3. Alerts Snapshot
   - Small list of most recent rules
   - CTA: "View all alerts"

4. Activity Feed (optional)
   - Based on SSE events

Waveform usage:
- Optional top hero background (active, subdued)
- Divider between sections

------------------------------------------------------------
2) ALERTS (SEARCH RULES)
------------------------------------------------------------

Route:
/alerts

Purpose:
Manage search-based alerts.

Layout:
- Header with title + "Create Alert" button
- Tab system:
  - Search Rules
  - Release Watchlist

Main content:
List of rules (cards or table hybrid)

Each rule item:
- Name
- Sources (icons)
- Poll interval
- Status (active/disabled)
- Last run / next run

Actions:
- Edit
- Disable
- Delete (hard)

Pagination:
- limit/offset → Load More button

Empty state:
- WaveTrace + "No alerts yet"

------------------------------------------------------------
3) ALERT EDITOR
------------------------------------------------------------

Route:
/alerts/new
/alerts/:id

Purpose:
Create or edit alert rules.

Layout:
- Form card centered (max width ~720px)

Fields:
- Name
- Sources (multi-select)
- Polling interval
- Filters (future)

Actions:
- Save
- Cancel
- Delete (if editing)

Validation:
- Map validation_error.details → fields

Waveform:
- NONE (keep clean)

------------------------------------------------------------
4) WATCHLIST (RELEASES)
------------------------------------------------------------

Route:
/watchlist

Purpose:
Track specific releases.

Layout:
- Similar to Alerts but focused on releases

Each item:
- Album name
- Artist
- Target price
- Condition
- Active toggle

Actions:
- Edit
- Disable
- Delete

------------------------------------------------------------
5) WATCHLIST ITEM EDITOR
------------------------------------------------------------

Route:
/watchlist/:id

Purpose:
Edit release tracking preferences.

Layout:
- Same as Alert Editor

Fields:
- Target price
- Condition
- Active toggle

------------------------------------------------------------
6) NOTIFICATIONS INBOX
------------------------------------------------------------

Route:
/notifications

Purpose:
View system events.

Layout:
- Vertical list (infinite scroll or load more)

Each item:
- Title (e.g. "Match found")
- Timestamp
- Linked entity (alert/release)
- Read/unread state

Actions:
- Mark as read

Realtime:
- SSE pushes new items
- Insert at top
- Update unread badge

Waveform:
- NONE in list
- Optional divider between sections

------------------------------------------------------------
7) INTEGRATIONS (DISCOGS)
------------------------------------------------------------

Route:
/integrations

Purpose:
Manage external account connections.

Sections:

1. Discogs Status Card
   - Connected / Not connected
   - CTA: Connect / Reconnect

2. Import Section
   - Buttons:
     - Import Wantlist
     - Import Collection
   - Shows job status

3. Import Progress
   - Poll /import/{job_id}
   - Show:
     - progress
     - counts
     - errors

------------------------------------------------------------
8) SETTINGS
------------------------------------------------------------

Route:
/settings

Tabs:
- Profile
- Account (danger zone)

Profile:
- Display name
- Preferences

Account:
- Logout button
- Deactivate account (danger zone)

Deactivate flow:
- Confirmation modal required
- After success → clear session + redirect

------------------------------------------------------------
9) APP SHELL (GLOBAL)
------------------------------------------------------------

Includes:
- Sidebar navigation
- Top bar (notifications + avatar)

Notification badge:
- Poll unread count
- Update on SSE

Toasts:
- On new match
- On errors

Waveform:
- Header band only (calm)

------------------------------------------------------------
10) EMPTY STATES
------------------------------------------------------------

Used in:
- Alerts
- Watchlist
- Notifications

Structure:
- Small WaveTrace (active, reduced size)
- Message
- CTA

------------------------------------------------------------
11) LOADING STATES
------------------------------------------------------------

- Skeleton loaders for lists
- Spinner for forms
- No waveform during loading

------------------------------------------------------------
12) ERROR STATES
------------------------------------------------------------

- Global banner for http_error
- Inline for validation_error
- Retry CTA where applicable

------------------------------------------------------------
END OF SPEC
